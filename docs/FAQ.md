# StoryForge FAQ

AI/LLM 파인튜닝이 처음인 사용자를 위한 자주 묻는 질문.

---

## 수집

### Q: 재생목록 전체를 한번에 수집해도 되나요?
A: 가능하지만, 영상이 수백 개 이상이면 YouTube가 일시적으로 요청을 차단(rate limit)할 수 있습니다. StoryForge는 영상 간 2초 딜레이를 자동으로 넣고, 차단 감지 시 60초 대기 후 재시도합니다. 1,000개 이상이면 수집에 40분~1시간 정도 걸릴 수 있습니다.

### Q: 수집 중에 멈추면 데이터가 날아가나요?
A: 아닙니다. 영상 하나가 완료될 때마다 즉시 저장됩니다. 중간에 멈추거나 에러가 나도 그때까지 수집된 데이터는 보존됩니다. 다시 시작하면 이미 수집한 영상은 자동으로 건너뛰고 나머지만 수집합니다.

### Q: 같은 영상을 두 번 수집하면 어떻게 되나요?
A: 영상 ID 기준으로 중복을 자동 감지합니다. 이미 수집한 영상은 건너뛰므로 데이터가 중복되지 않습니다. 단일 URL로 수집한 영상이 재생목록에도 포함되어 있어도 괜찮습니다.

### Q: "상위 10%" 필터는 뭔가요?
A: 재생목록의 영상을 조회수 기준으로 정렬한 뒤, 상위 N%만 수집하는 옵션입니다. 다른 채널의 대량 영상에서 검증된 퀄리티의 콘텐츠만 골라 학습하고 싶을 때 유용합니다. 같은 채널에서 스타일이 일정하다면 전체를 수집하는 게 데이터 다양성 면에서 더 좋습니다.

## 정제

### Q: 정제가 뭘 하는 건가요?
A: 수집된 텍스트를 AI가 학습할 수 있는 형태로 변환하는 과정입니다:
1. 텍스트를 적절한 길이(~1,500자)로 분리 (청킹)
2. 음성 인식 오타를 자동 교정 (STT 교정)
3. 각 장면의 장르/핵심사건/감정흐름/서사기법 등을 분석
4. 4가지 유형의 학습 데이터를 자동 생성

### Q: 4-Task 학습 데이터가 뭔가요?
A: 모델이 긴 스토리를 쓸 수 있도록 4가지 능력을 각각 가르치는 데이터입니다:
- **Task 1 (아웃라인)**: 전체 스토리 구조를 기획하는 법
- **Task 2 (장면 확장)**: 앞 내용을 이어받아 특정 장면을 쓰는 법
- **Task 3 (연속 집필)**: 직전 장면에서 자연스럽게 이어쓰는 법
- **Task 4 (스타일)**: 특정 분위기/장르에 맞는 문체로 쓰는 법

### Q: 정제에 얼마나 걸리나요?
A: 에피소드당 약 75초 (교정 + 분석 2-pass). 10개 에피소드는 ~12분, 1,000개는 ~20시간입니다. 백그라운드로 실행되므로 기다리는 동안 다른 작업이 가능합니다.

### Q: STT 오타 교정이 완벽하지 않던데요?
A: 음성 인식 오류 중 명확한 것(동음이의어, 받침 오류)은 자동 교정하지만, 맥락에 따라 판단이 어려운 것은 원문을 유지합니다. 오타가 일부 남아있어도 학습에는 큰 영향이 없습니다. 모델이 배우는 건 개별 단어가 아니라 스토리의 구조와 흐름입니다.

### Q: 방송 인트로/아웃트로도 학습되나요?
A: 아닙니다. 정제 과정에서 "구독 부탁드립니다", "다음 시간에 만나요" 같은 비스토리 콘텐츠를 자동으로 감지하고 학습 데이터에서 제외합니다.

## 학습

### Q: 어떤 모델을 선택해야 하나요?
A: GPU VRAM에 따라 선택하세요:
| 모델 | VRAM 필요 | 권장 GPU |
|------|----------|---------|
| Gemma 4 8B | ~12GB | RTX 3080 이상 |
| Gemma 4 12B | ~18GB | RTX 4080 이상 |
| Gemma 4 27B | ~24GB+ | RTX 5090, A100 이상 |

일반적으로 **VRAM이 허락하는 한 큰 모델**이 결과가 좋습니다. 12B는 가성비가 좋고, 27B는 최고 품질이지만 24GB에서는 빠듯합니다.

### Q: LoRA(QLoRA)가 뭔가요?
A: 기존 AI 모델의 핵심은 그대로 두고, 작은 "어댑터"만 추가로 학습하는 기법입니다. 모델 전체를 학습하는 것보다 메모리가 훨씬 적게 들고, 원래 모델의 능력을 유지하면서 새로운 스타일을 배울 수 있습니다.

### Q: Full Fine-tuning이랑 뭐가 다른가요?
A: 
| | QLoRA | Full Fine-tuning |
|---|---|---|
| 바꾸는 것 | 어댑터(~1%) | 모델 전체 |
| VRAM | 12~24GB | 32~80GB+ |
| 원본 능력 유지 | O (일반 지식 보존) | X (학습 데이터에 특화) |
| 새로운 소재 확장 | 잘됨 | 약해질 수 있음 |

QLoRA는 원본 모델의 능력 위에 새 스킬을 "얹는" 것이고, Full FT는 모델을 "갈아엎는" 것입니다. 다양한 소재로 창의적인 글을 쓰고 싶다면 QLoRA가 유리합니다.

### Q: 학습 데이터가 많을수록 좋은가요?
A: 네, 일반적으로 그렇습니다. 하지만 "품질 > 수량"입니다. 500개의 깨끗한 데이터가 5,000개의 노이즈 데이터보다 낫습니다. StoryForge는 자동 분석과 필터링으로 데이터 품질을 보장합니다. 50,000개 이상이면 LoRA 학습에 매우 충분합니다.

### Q: 여러 채널의 데이터를 합쳐서 학습해도 되나요?
A: **합쳐서 한번에 학습하는 게 좋습니다.** 다양한 소재/구조를 동시에 보면 모델이 "공통 패턴(=스토리 구조)"을 더 잘 추출합니다. 채널별로 따로 학습하거나 추가학습(continual training)하면 이전 학습을 잊어버릴 수 있습니다.

### Q: 학습 파라미터는 뭘 의미하나요?
A: 기본값으로 시작하면 됩니다. 참고로:
- **학습률 (Learning Rate)**: 모델이 얼마나 빠르게 배우는지. 너무 높으면 불안정, 너무 낮으면 느림. 기본 0.0001.
- **에포크 (Epochs)**: 전체 데이터를 몇 번 반복해서 볼지. 너무 많으면 "암기"(오버피팅). 기본 2.
- **LoRA 랭크**: 어댑터의 크기. 클수록 더 많이 배우지만 메모리 사용 증가. 기본 32.
- **배치 사이즈**: 한번에 처리하는 데이터 수. 메모리에 맞게 설정. 기본 2.
- **시퀀스 길이**: 한번에 읽는 텍스트 길이. 길수록 긴 문맥을 이해하지만 메모리 증가. 기본 4096.

### Q: 학습에 얼마나 걸리나요?
A: 데이터 양과 모델 크기에 따라 다릅니다. 대략:
- 420개 데이터, 12B 모델, 2 에포크: ~30분~1시간
- 50,000개 데이터, 12B 모델, 2 에포크: ~수 시간
첫 실행 시 Unsloth 설치 + 모델 다운로드에 추가 10~20분 소요됩니다.

### Q: Ollama에 있는 모델로 바로 학습할 수는 없나요?
A: 없습니다. Ollama 모델(GGUF)은 추론(생성)용이고, 학습은 별도의 HuggingFace 모델(safetensors)이 필요합니다. 학습 완료 후 자동으로 GGUF 변환 → Ollama 등록까지 처리됩니다.

## 생성

### Q: 학습한 모델로 어떻게 스크립트를 만드나요?
A: 생성 탭 → 스크립트 모드에서:
1. 장르와 주제를 입력
2. 아웃라인(전체 구조)이 자동 생성됨 → 확인/수정
3. 승인하면 장면별로 순차 생성
4. 각 장면을 "다시 쓰기"하거나 직접 편집 가능
5. 완료 후 전체 내보내기

### Q: 학습 안 한 모델로도 생성할 수 있나요?
A: 네. 기본 gemma4 모델로도 생성 가능합니다. 다만 학습된 모델이 해당 장르의 문체와 구조를 훨씬 잘 살립니다.

### Q: 생성된 스크립트가 학습 데이터를 그대로 베끼지는 않나요?
A: LoRA 학습은 특정 텍스트를 암기하는 게 아니라 패턴(구조, 문체, 전개 방식)을 학습합니다. 생성 시 새로운 주제를 입력하면 학습된 구조 위에 새로운 내용을 만들어냅니다. 완전히 동일한 텍스트가 나올 가능성은 거의 없습니다.
